ORDEM ASSINTÓTICA

A ordem assintótica é uma ferramenta usada em ciência da computação para comparar a eficiência de algoritmos. Ela permite que ignoremos as diferenças de eficiência que são irrelevantes para grandes entradas.

Para entender a ordem assintótica, precisamos primeiro entender o que significa "crescimento assintótico". O crescimento assintótico de uma função f(n) é o seu comportamento a medida que n tende ao infinito.

Por exemplo, a função f(n) = n^2 cresce assintoticamente mais rápido que a função g(n) = n. Isso significa que, para grandes valores de n, f(n) será muito maior que g(n).

A ordem assintótica de uma função é representada por uma letra maiúscula. As letras mais comuns são O, Ω e Θ.

  • O(g(n)) significa que f(n) está na ordem de no máximo g(n). Isso significa que existe uma constante c tal que f(n) ≤ c * g(n) para grandes valores de

  • Ω(g(n)) significa que f(n) está na ordem de no mínimo g(n). Isso significa que existe uma constante c tal que f(n) ≥ c * g(n) para grandes valores de n.

  • Θ(g(n)) significa que f(n) está na ordem de g(n). Isso significa que existem constantes c1 e c2 tais que c1 * g(n) ≤ f(n) ≤ c2 * g(n) para grandes valores de n.

Por exemplo, a ordem assintótica do tempo de execução do algoritmo de ordenação por inserção é O(n^2). Isso significa que, para grandes entradas, o tempo de execução do algoritmo será proporcional ao quadrado do tamanho da entrada.

A ordem assintótica é uma ferramenta importante para comparar a eficiência de algoritmos. Ela permite que ignoremos as diferenças de eficiência que são irrelevantes para grandes entradas.

Aqui estão alguns exemplos de ordens assintóticas comuns:

  • OO(1): Constante

  • O(log n): Logarítmica

  • O(n): Linear

  • O(n log n): Logarítmica quadrática

  • O(n^2): Quadrada

  • O(n^3): Cúbica

  • O(n^k): Polinomial

  • O(2^n): Exponencial

O que é a complexidade pessimista ?


A complexidade pessimista de um algoritmo é a ordem assintótica do tempo de execução do algoritmo na pior entrada possível.

Para calcular a complexidade pessimista de um algoritmo, precisamos identificar a pior entrada possível para o algoritmo. Essa entrada é aquela que leva o algoritmo a executar o maior número possível de operações.

Por exemplo, a complexidade pessimista do algoritmo de ordenação por inserção é O(n^2). Isso significa que, para a pior entrada possível, o tempo de execução do algoritmo será proporcional ao quadrado do tamanho da entrada.

A complexidade pessimista é uma ferramenta útil para comparar a eficiência de algoritmos. Ela nos permite identificar os algoritmos que são mais suscetíveis a entradas ruins.

Aqui estão alguns exemplos de complexidades pessimistas comuns:

  • O(1): Constante

  • O(log n): Logarítmica

  • O(n): Linear

  • O(n log n): Logarítmica quadrática

  • O(n^2): Quadrada

  • O(n^3): Cúbica

  • O(n^k): Polinomial

  • O(2^n): Exponencial

A complexidade pessimista é um limite inferior para o tempo de execução de um algoritmo. Isso significa que, para qualquer entrada, o tempo de execução do algoritmo nunca será maior que sua complexidade pessimista.

No entanto, a complexidade pessimista pode ser uma estimativa muito conservadora do tempo de execução real de um algoritmo. Isso ocorre porque a pior entrada possível pode ser muito improvável de ocorrer na prática.

Para obter uma estimativa mais precisa do tempo de execução real de um algoritmo, podemos usar a complexidade média ou a complexidade esperada.

O que é um Algoritmo Guloso ?


Algoritmo Guloso é uma solução comum para problemas de otimização onde:

  • Realiza a escolha que parece ser a melhor no momento na esperança de que a mesma acarrete em uma solução ou prevenção de futuros problemas a nível global.

  • É simples e de fácil implementação;

  • É míope: ele toma decisões com base nas informações disponíveis na iteração corrente.

Características:


  • Jamais se arrepende de uma decisão, as escolhas realizadas são definitivas;

  • Não leva em consideração as consequências de suas decisões;

  • Podem fazer cálculos repetitivos;

  • Nem sempre produz a melhor solução (depende da quantidade de informação fornecida);

  • Quanto mais informações, maior a chance de produzir uma solução melhor.

Algoritmos Gulosos vs Programação Dinâmica


  • Em um algoritmo de programação dinâmica a escolha pode depender da solução dos subproblemas, enquanto um algoritmo guloso vai tentar escolher a melhor solução naquele momento.

  • A solução dos problemas na programação dinâmica parte de baixo para cima, enquanto um algoritmo guloso vai de cima para baixo, ou seja, na programação dinâmica, as soluções para todos os subproblemas são calculadas partindo dos menores subproblemas para os maiores.

  • Os resultados dos subproblemas na programação dinâmica são salvos, facilitando a prova de correção.

Pontos a favor


  • Simples e de fácil implementação;

  • Algoritmos de rápida execução;

  • Podem fornecer a melhor solução (estado ideal).


Pontos Contra


  • Dependentes de informações;

  • Correm o risco de entrarem em loops infinitos;

  • Situacionais, só resolvem tipos específicos de problemas;

  • Decisões tomadas são irrevogáveis;

DIVIDIR PARA CONQUISTAR


A estratégia de dividir para conquistar, ou divide et impera, é uma técnica de resolução de problemas que consiste em dividir um problema maior em problemas menores, mais fáceis de resolver.

Essa técnica é frequentemente usada em ciência da computação para resolver problemas computacionais complexos.

O algoritmo de ordenação mergesort é um exemplo de algoritmo que usa a estratégia de dividir para conquistar. O mergesort divide o vetor a ser ordenado em dois vetores menores, ordena esses vetores e, em seguida, mescla os vetores ordenados para obter o vetor ordenado final.

Outro exemplo de algoritmo que usa a estratégia de dividir para conquistar é o algoritmo de busca binária. A busca binária divide o intervalo de busca em duas metades, verifica se o elemento procurado está na metade esquerda ou na metade direita e, em seguida, repete o processo na metade apropriada.

A estratégia de dividir para conquistar é uma técnica eficiente para resolver problemas computacionais complexos. Ela pode ser usada para resolver uma ampla gama de problemas, incluindo:

  • Problemas de ordenação

  • Problemas de busca

  • Problemas de otimização

  • Problemas de grafos

Aqui estão algumas vantagens da estratégia de dividir para conquistar:


  • É uma técnica eficiente para resolver problemas complexos.

  • É uma técnica relativamente fácil de implementar.

No entanto, a estratégia de dividir para conquistar também tem algumas desvantagens:


  • Pode ser ineficiente para problemas pequenos.

  • Pode exigir uma quantidade significativa de memória.

P, NP E NP-COMPLETO


As classes P, NP e NP-completo são classes de problemas de decisão.

P é a classe de problemas de decisão que podem ser resolvidos em tempo polinomial. Isso significa que, para qualquer entrada de tamanho n, o algoritmo que resolve o problema pode executar no máximo n^k operações, onde k é uma constante.


NP é a classe de problemas de decisão que podem ser verificados em tempo polinomial. Isso significa que, para qualquer entrada de tamanho n, é possível verificar se a resposta do problema é correta em tempo polinomial.

NP-completo é uma classe de problemas de decisão que são NP e que, se houver um algoritmo polinomial para resolver qualquer problema NP, então também haverá um algoritmo polinomial para resolver todos os problemas NP.

Alguns exemplos de problemas NP-completos incluem:

  • O problema do caixeiro viajante: Encontre o circuito mais curto que visita todas as cidades.

  • O problema do knapsack: Encontre o conjunto de itens mais valioso que pode ser colocado em uma mochila de tamanho limitado.

  • O problema do SAT: Determine se uma fórmula booleana é satisfeita.


A questão de saber se P = NP é um dos problemas mais importantes em ciência da computação. Se P = NP, então todos os problemas NP podem ser resolvidos em tempo polinomial. Isso teria implicações profundas para muitas áreas da ciência e da engenharia.

No entanto, se P ≠ NP, então alguns problemas NP são inerentemente difíceis. Isso significa que, para esses problemas, não existe um algoritmo polinomial que possa resolver todos os casos.

Até o momento, a questão de saber se P = NP ainda não foi resolvida.
